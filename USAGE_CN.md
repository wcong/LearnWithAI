## 1. 用户注册与登录

### 注册

在登录页面点击**注册**链接，输入用户名和密码：
- 用户名：至少 2 个字符
- 密码：至少 4 个字符

### 登录

输入凭证登录后获得 JWT Token。Token 保存在 `localStorage` 中，刷新页面不会丢失。

### 自动登录历史

每次登录会记录 IP 地址（脱敏处理）、User-Agent 和地理位置（通过 ip-api.com 查询）。

## 3. 学习领域管理

**页面**：`/domain`（三栏布局：树 + 对话 + 工具）

### 创建领域

点击左侧面板顶部的 **➕ 新建领域**，输入名称和可选的描述。

### 浏览知识树

- 知识树**横向**向右展开
- 点击节点选中该领域，右侧聊天面板和工具面板随之激活
- 点击 **➕ 添加子领域** 创建分支节点，支持无限层级下钻

  示例：`机器学习 → 监督学习 → 决策树 → 随机森林`

### 编辑 / 删除

- **右键**或使用节点的上下文菜单进行重命名或删除
- 删除父节点时会递归删除所有子节点

### AI 子领域审查

点击 **🔍 审查** 触发 AI 代理执行以下流程：
1. 列出所有子领域及分析状态
2. 递归为未分析的子领域生成分析
3. 聚合生成父领域的分析报告
4. 提示缺失的重要子方向

同时提供流式（SSE）版本，可实时查看 AI 的思考过程。

### 生成子领域建议

点击 **✨ 生成**，AI 会根据聊天记录分析，推荐 3–6 个可供深入研究的子方向。

### 润色描述

生成或编辑子领域后，点击 **🎨 润色**，AI 会优化描述文本但不改变标题。

## 4. AI 对话

**页面**：`/domain`（右侧面板）

### 开始对话

选中一个领域节点，在聊天输入框中输入问题并回车。

**AI 导师会：**
- 评估你当前的知识水平
- 引导探索核心概念
- 每次回复推荐 2–3 个下一步探索方向

### 上下文继承

AI 可以访问**所有祖先领域**的聊天历史，提供跨越整个知识树的上下文感知回答。

### 流式对话

使用**流式模式**可通过 SSE 实时逐 token 看到 AI 的回复过程。

### 技能模板注入

聊天时可以从下拉菜单中选择**技能模板**，让 AI 按特定角色/格式回答。  
例如内置的**"面试准备"**技能会让 AI 按面试准备材料格式输出。

### 聊天历史

所有消息持久化到数据库，重新打开页面时历史自动加载。

### 学习会话

点击 **💾 保存会话** 保存当前学习总结。可以在会话列表查看某个领域的所有保存记录。

### 删除消息

可以删除单条聊天消息（同时清除该领域的 agent 缓存）。

## 5. 学习笔记

**页面**：`/notes`（两栏布局：领域树 + 富文本编辑器）

### 编辑器功能

- 基于 **Quill.js** 富文本编辑器
- 支持标题、加粗、斜体、列表、代码块、图片等格式
- 内容以 **HTML 格式**按领域存储

### 自动保存 & RAG 索引

保存笔记时将触发后台任务：
1. 存储 HTML 内容到 `area_notes` 表
2. 后台线程**自动重建该领域的 RAG 索引**：
   - 清洗 HTML → 提取纯文本
   - 分块（每块 500 字符，重叠 50）
   - 生成向量嵌入（使用已配置的 LLM 提供商）
   - 存储到 `note_embeddings` 表

### 搜索笔记

参见 [RAG 语义搜索](#8-rag-语义搜索)。

## 6. Plan Mode — 自动学习规划

**页面**：`/plan`（实验功能）

Plan Mode 自动探索一个领域并生成结构化的学习路径。

### 使用方法

1. 访问 `/plan`（或首页点击 **🧠 学习计划**）
2. 输入领域名称（如"Transformer 架构"、"量子计算"）
3. 设置**最大探索深度**（默认 2）
4. 点击**开始**

### 工作流程

AI 递归执行以下步骤：
1. 在你的知识树中创建根领域节点
2. 生成领域概况（800–1500 字）
3. 提取 3–5 个具体可研究的子方向
4. 为每个子方向创建子节点
5. 对每个子节点递归执行步骤 2–4
6. 同层子领域**并发**探索

界面实时显示 SSE 事件：
- **area_created** — 新建领域节点
- **message** — AI 生成的概况内容
- **progress** — 当前探索深度与状态
- **thinking** — 实时 AI 思考过程

### 结果

探索后的领域会出现在知识树中，AI 生成的概况会保存为聊天消息。

## 7. 技能模板

**页面**：`/static/skills.html`

技能模板是可复用的提示词模板，控制 AI 在聊天中的回答方式。

### 技能类型

| 类型 | 说明 |
|------|------|
| **个人技能** | 由你创建，仅自己可见 |
| **全局技能** | 管理员创建，对所有用户可见 |
| **默认技能** | 系统内置（如"面试准备"），不可删除 |

### 创建个人技能

1. 点击 **➕ 新建技能**
2. 输入**名称**和**描述**
3. 编写**提示词模板**，用 `{topic}` 作为用户输入的占位符
4. 保存 — 该技能即可在聊天下拉菜单中使用

### 内置默认技能：面试准备

系统自动创建"面试准备"技能模板，激活后 AI 会按以下结构回答：
- 核心概念与原理
- 实现细节与架构设计
- 实际应用场景
- 常见面试问题与参考答案

### 管理员：管理全局技能

管理员用户可以创建、编辑和删除对所有用户可见的全局技能。

## 8. RAG 语义搜索

RAG（检索增强生成）允许你用自然语言搜索所有笔记内容。

### 搜索原理

1. 查询文本被转换为**向量嵌入**（通过已配置的 LLM 提供商）
2. 数据库中所有笔记分块通过**余弦相似度**匹配
3. 返回 Top-K 结果及相似度评分
4. **降级方案**：嵌入不可用时自动使用关键词匹配

### 访问方式

RAG 搜索可通过聊天界面调用，或通过 `POST /api/rag/search` API 直接查询。

## 9. 管理员面板

**接口**：`GET /api/admin/stats`（需要 JWT Token，仅管理员可访问）

管理员面板提供：
- **全局汇总**：总用户数、总领域数、总消息数、Token 用量
- **按用户分解**：每个用户的领域数、消息数、Token 消耗（提示/补全/总计）

通过 `ADMIN_USERNAME` 环境变量配置管理员用户名。

## 10. 移动端界面

移动端提供触屏优化的功能版本：

| 页面 | 地址 | 说明 |
|------|------|------|
| 首页 | `/mobile` | 移动端首页 |
| 领域 | `/mobile/domain` | 对话 + 覆盖层领域导航 |
| 笔记 | `/mobile/notes` | 移动端笔记编辑器 |
| 计划 | `/mobile/plan` | 移动端 Plan Mode |

### 导航模式

移动端页面采用**滑出面板**模式：
- 主要内容（对话、笔记、计划）填满屏幕
- 点击按钮滑出领域树覆盖层
- 选择领域切换主要内容

所有功能（技能对话、笔记编辑、Plan Mode、树导航）与桌面端一致。
